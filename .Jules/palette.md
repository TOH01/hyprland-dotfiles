## 2023-11-20 - Adding Accessibility and Keyboard Support to QML Buttons
**Learning:** In Qt Quick (QML), custom UI components like Buttons that are built directly from `Item` or `Rectangle` do not have implicit keyboard navigation (`activeFocusOnTab`), visual focus states, or screen reader visibility. Standard `TapHandler` only reacts to pointers, so `Keys` handlers must be explicitly added to support Enter/Space activation, and `Accessible` properties must be declared for screen reader compliance.
**Action:** Always ensure custom QML interactive elements implement `activeFocusOnTab: true`, have a visual focus indicator, handle Enter/Space key events, and export an `Accessible.role` along with `Accessible.name`. For icon-only buttons, supply an explicit `accessibleName` property.
